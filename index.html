Design Policy

Super-Mjograph is designed in a way I find easy-to-use. Therefore, if you agree with my thoughts below, this piece of software would be a great research partner for you too!
Existing tools are too command-oriented
No more exhaustion in looking up commands. The GUI is simple and intuitive, and takes less time to learn.

That said, CUI is inevitable
Super-Mjograph is scriptable with Applescript (although currently minimal). Python interface is also available.

Visualization should be prompt and expeditious
Visualization is a basic routine for every researcher, and therefore it should be done as efficiently as possible. Imagine how many mouse clicks we need in Excel even for a very simple line plot.

Plot is an art where a researcher's aesthetics can be sensed
What if customizing appearance is too laborious? We are then discouraged from beautifying the plot. Super-Mjograph's well-organized interface encourages you to express your aesthetics.

Stable user interface
An unnecessary update of the UI is very annoying. I continue developing Super-Mjograph with deliberate consideration given to interface changes.

Mac apps should be sexy
I love Mac and its unified look-and-feel. Cross-platform apps are useful, but they sadly mar the unity of appearance.
